Django Default Template Engine (DTL)

1. How the Django Template Engine Works

Django uses the Django Template Language (DTL) to separate presentation (HTML) from business logic (Python code).

  1. A view prepares a context dictionary.
  2. The view calls render(request, "template.html", context).
  3. The template engine processes template tags and variables.
  4. The final HTML is sent to the browser.

2. Displaying Context Variables

Context variables are displayed using double curly braces:

{{ username }}

Example view context:

context = {
    "username": "Amine"
}

This will render as:

Amine

3. Blocks (Template Inheritance)

Blocks are used with template inheritance to allow child templates to override parts of a base template.

Base Template:

{% block content %}
Default content
{% endblock %}

Child Template:

{% extends "base.html" %}

{% block content %}
My custom page content
{% endblock %}

4. Loops (for ... in)

Used to iterate over lists or querysets:

{% for task in tasks %}
    {{ task }}
{% endfor %}

You can also handle empty lists:

{% for task in tasks %}
    {{ task }}
{% empty %}
    No tasks found.
{% endfor %}

5. If Control Structures

Django templates support conditional rendering:

{% if user.is_authenticated %}
    Welcome {{ user.username }}
{% else %}
    Please login
{% endif %}

You can also use:

  • elif
  • and
  • or
  • not

Important Concept

Django templates are intentionally limited. They should only contain presentation logic (loops, conditions, rendering). Business logic must remain in views and models.
