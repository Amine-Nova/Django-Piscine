Process of Displaying a Static Web Page

1. Client Request

The process starts when a client (usually a web browser) sends an HTTP request to the server by typing a URL or clicking a link.

2. URL Routing

The web server forwards the request to the web framework. The router matches the requested URL to a specific view.

3. View Execution

The view handles the request. For a static page, it performs minimal logic and selects the appropriate template to display.

4. Template Rendering

The view sends the template to the template engine. Since the page is static, the template is rendered with little or no dynamic data, producing plain HTML.

5. Response Creation

The generated HTML is wrapped inside an HTTP response, including headers and a status code.

6. Server Response

The server sends the HTTP response back to the client.

7. Client Display

The browser receives the response, parses the HTML, and displays the static web page to the user.

Summary

Request → View → Template → HTML → Response → Browser Display
